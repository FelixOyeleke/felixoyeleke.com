Mission

PySide performance pass

The desktop client ships with PySide so UI iterations stay fast, but hot paths now live inside native C++ modules that expose Qt-friendly hooks. Those modules stream buffers straight from the Rust parser into GPU textures, eliminating the copy overhead that used to block the event loop.

  • Render loop rewritten in C++ (/O2 on Windows, Clang on macOS) and invoked through Qt signals.
  • Shared-memory bridge keeps PySide widgets in sync with GPU textures without starving the main thread.
  • Optional CUDA/OpenCL kernels accelerate palette transforms and measurement overlays when a capable GPU is available.

Optimization snapshot

  • Done 48% faster PySide repaint cycle after moving interpolation to C++.
  • Done GPU shaders collapse palette swaps to <10 ms even on mid-tier cards.
  • In progress Unified profiler overlay comparing CPU vs. GPU workloads in real time.

Waterfall view (shipping)

We now render high-density waterfalls with configurable palettes (amplitude, gradient, adaptive gain). Parsed packets stay in GPU memory, so scrubbing through 60 GB archives feels instant. Cursor sampling exposes depth, slant range, and ping IDs without leaving the viewer.

  • Packet-to-pixel pipeline built in Rust + wgpu.
  • Telemetry overlay renders layback, GPS, and INS drift per ping.
  • Export path saves annotated frames as GeoTIFF previews.

GUI workspace (in development)

The next milestone is a PySide-based shell backed by the same Rust core. It will bundle a map window, waterfall panel, and telemetry console so operators can cross-reference anomalies instantly while the C++ layer streams GPU frames into Qt widgets.

  • Sprint Integrate WebGL canvas for panning/zooming.
  • Sprint Build measurement + annotation overlays.

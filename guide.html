Owner Guide

How to update content, add works and posts, refresh the cache for visitors, and deploy changes.

1) Update “My Works” on the homepage

  • Open index.html.
  • Find the section comment .
  • Each work item is a div.project-item with date, title link, description, and image.
  • Edit the link text to your new work (e.g., “Coursera Course”, “Udemy Course”, “Documentary Video”).
  • Swap the image src with a file in /images (keep SVGs or JPGs).

2) Add or edit blog posts

  • Blog listing is at blog.html.
  • Each post is an <article class="blog-post"> with meta, title, tags, and excerpt.
  • To add a new post page, copy an existing file under /blog/ and update the content.
  • Link to the new page from blog.html by adding a new article.blog-post block.

3) Update colors and typography

  • Design tokens are in css/main-styles.css under :root.
  • Accent black variables: --gold and --gold-dark currently set to softer black.
  • Change spacing, fonts, and sizes via the --font-* and --spacing-* variables.

4) Cache busting for visitors (Service Worker)

If visitors don’t see updates, increase the cache version and refresh.

  1. Open sw.js.
  2. Change const CACHE_NAME = 'felix-oyeleke-vX' to the next number (e.g., v3).
  3. Optionally add new files to urlsToCache (CSS, images, pages).
  4. Save, deploy, then reload the site. On desktop, do a hard refresh (Cmd/Ctrl+Shift+R).

5) Deployment

  • This is a static site (HTML/CSS/JS). Host on Netlify or any static host.
  • Upload the folder or push to your connected repo. Netlify will serve directly (no build step).

6) Troubleshooting

  • Update not showing? Bump CACHE_NAME in sw.js and hard refresh.
  • Broken link? Confirm the href points to an existing file and the path is correct.
  • Images not loading? Ensure the file is in /images and the name matches exactly.
